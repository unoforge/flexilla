`@flexilla/autocomplete` builds on top of `@flexilla/select-core` and connects the search input, option list, and selected values in the DOM.

Use it when you want:

- typed search over a list of options
- headless DOM bindings without a framework dependency
- single or multiple selection

## Markup structure

The root must have an `id`. For the class-based API, add `data-fx-autocomplete`.

```html
<div id="framework-autocomplete" data-fx-autocomplete>
  <input
    type="text"
    data-select-input
    data-autocomplete-id="framework-autocomplete"
    placeholder="Search frameworks"
  />

  <div
    data-select-content
    data-select-id="framework-autocomplete"
    data-placement="bottom-start"
    class="ui-popper"
    hidden
  >
    <button data-select-item="astro">Astro</button>
    <button data-select-item="vue">Vue</button>
    <button data-select-item="laravel">Laravel</button>
    <button data-select-item="symfony">Symfony</button>
  </div>

  <div data-selected-value data-select-id="framework-autocomplete"></div>
  <button data-select-clear data-select-id="framework-autocomplete">Clear</button>
</div>
```

Useful overlay attributes on `data-select-content`:

- `data-placement`
- `data-offset-distance`
- `data-prevent-close-outside`
- `data-prevent-close-inside`
- `data-readjust-height`
- `data-min-height`

`readjustHeight` is enabled by default for autocomplete and can be disabled with `data-readjust-height="false"` or `{ readjustHeight: false }`.

## Basic usage

### Class API

```js
import { Autocomplete } from "@flexilla/autocomplete";

const autocomplete = new Autocomplete("#framework-autocomplete");
```

### Auto init

```js
import { Autocomplete } from "@flexilla/autocomplete";

Autocomplete.autoInit();
```

### Controller API

```js
import { createAutocomplete } from "@flexilla/autocomplete";

const root = document.querySelector("#framework-autocomplete");
const autocomplete = createAutocomplete();
const { destroy } = autocomplete.connect({ root });
```

## Custom filtering

You can provide your own filter function.

```js
import { createAutocomplete } from "@flexilla/autocomplete";

const autocomplete = createAutocomplete({
  filter: (query, item) => {
    return item.label?.toLowerCase().startsWith(query.toLowerCase()) ?? false;
  },
});
```

## Multiple selection

```js
const autocomplete = new Autocomplete("#framework-autocomplete", {
  multiple: true,
});
```

You can also declare it in markup:

```html
<div id="framework-autocomplete" data-fx-autocomplete data-multiple>
  ...
</div>
```

For multiple autocomplete instances, clicks inside the list stay open by default so the user can keep selecting values.

## State and methods

Like `select`, the controller exposes the `select-core` methods directly:

- `open()`
- `close()`
- `toggle()`
- `select(value)`
- `unselect(value)`
- `clear()`
- `toggleValue(value)`
- `highlight(index)`
- `highlightNext()`
- `highlightPrev()`
- `setSearch(query)`
- `getState()`
- `subscribe(listener)`

Example:

```js
autocomplete.subscribe((state) => {
  console.log(state.search, state.selectedValues);
});
```

## Lifecycle

The class API exposes `cleanup()`:

```js
const autocomplete = new Autocomplete("#framework-autocomplete");

// Later
autocomplete.cleanup();
```

The controller API still returns a `destroy()` function:

```js
const { destroy } = autocomplete.connect({ root });

// Later
destroy();
```

That is especially useful in frameworks where the DOM subtree can be replaced or unmounted.

## When to use Autocomplete vs Select

- Use autocomplete when the user should filter options by typing.
- Use [Select](/docs/components/select) when the options can stay visible without search.
