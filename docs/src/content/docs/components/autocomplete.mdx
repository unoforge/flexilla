`@flexilla/autocomplete` builds on top of `@flexilla/select-core` and connects the search input, option list, and selected values in the DOM.

Use it when you want:

- typed search over a list of options
- single or multiple selection
- rich selected content or tags
- a headless DOM binding without a framework dependency

## Markup structure

`autocomplete` does not need a wrapper. The input, content, and selected-value container only need to share the same id.

```html
<input
  data-fx-autocomplete
  data-select-input
  data-autocomplete-id="framework-autocomplete"
  type="text"
  placeholder="Search frameworks"
/>

<div
  data-select-content
  data-select-id="framework-autocomplete"
  data-placement="bottom-start"
  class="ui-popper"
  hidden
>
  <button data-select-item="astro">Astro</button>
  <button data-select-item="vue">Vue</button>
  <button data-select-item="laravel">Laravel</button>
</div>

<div data-selected-value data-select-id="framework-autocomplete"></div>
<button data-select-clear data-select-id="framework-autocomplete">Clear</button>
```

Common elements:

- `data-select-input`
- `data-select-content`
- `data-select-item`
- `data-selected-value`
- `data-select-clear`
- `data-select-clear-all`
- `data-select-remove`

Useful overlay attributes on `data-select-content`:

- `data-placement`
- `data-offset-distance`
- `data-prevent-close-outside`
- `data-prevent-close-inside`
- `data-readjust-height`
- `data-min-height`

The panel receives `--trigger-width`, so you can size it from the input width:

```css
[data-select-content] {
  width: var(--trigger-width);
}
```

`readjustHeight` is enabled by default for autocomplete. Disable it with `data-readjust-height="false"` or `{ readjustHeight: false }`.

## Basic usage

### Class API

```js
import { Autocomplete } from "@flexilla/autocomplete";

const autocomplete = new Autocomplete('[data-select-input][data-autocomplete-id="framework-autocomplete"]');
```

You can also pass the input element directly.

The constructor now expects a real selector or a real `HTMLElement`. It reads the id from that element and resolves the rest of the autocomplete from there.

### Auto init

```js
import { Autocomplete } from "@flexilla/autocomplete";

Autocomplete.autoInit();
```

`Autocomplete.autoInit()` looks for `[data-fx-autocomplete]`, which is usually the input element.

### Controller API

```js
import { createAutocomplete } from "@flexilla/autocomplete";

const autocomplete = createAutocomplete();
const { destroy } = autocomplete.connect({
  element: '[data-select-input][data-autocomplete-id="framework-autocomplete"]',
});
```

You can also connect with an existing element:

```js
const input = document.querySelector('[data-autocomplete-id="framework-autocomplete"]');
autocomplete.connect({ element: input });
```

## Search and filtering

Autocomplete uses the input as the main trigger. The panel opens on focus or click, and filtering happens as the user types.

You can customize the filter function:

```js
import { createAutocomplete } from "@flexilla/autocomplete";

const autocomplete = createAutocomplete({
  filter: (query, item) => {
    return item.label?.toLowerCase().startsWith(query.toLowerCase()) ?? false;
  },
});
```

## Custom selected templates

`data-selected-value` can define a custom selected model:

- `template[data-selected-model]`
- `data-bind`
- `data-bind-src`
- `data-bind-alt`
- `data-bind-title`
- `data-bind-href`
- `data-bind-style`
- `data-select-label`
- `data-select-value`
- `data-select-remove`

```html
<div data-selected-value data-select-id="owners">
  <template data-selected-model>
    <span class="inline-flex items-center gap-2">
      <span
        class="size-5 rounded-full"
        data-bind-style="badgeStyle"
        data-bind="initials"
      ></span>
      <span data-bind="label"></span>
    </span>
  </template>
  <span data-placeholder>No owner selected</span>
</div>

<li
  data-select-item="brenda"
  data-label="Brenda Stone"
  data-initials="BS"
  data-badge-style="background-color:#fecdd3;color:#881337"
>
  ...
</li>
```

## Multiple selection

Enable multiple mode with options:

```js
const autocomplete = new Autocomplete('[data-select-input][data-autocomplete-id="team-members"]', {
  multiple: true,
});
```

Or in markup:

```html
<input
  data-fx-autocomplete
  data-multiple
  data-select-input
  data-autocomplete-id="team-members"
  type="text"
/>
```

In multiple mode:

- selecting items keeps the panel open
- removing a selected item also keeps the panel open
- the input remains available for continued search

## Empty state

Use `data-select-empty` to render a message when no item matches the query.

```html
<template data-select-empty>
  <div>
    No owner found for "<span data-select-empty-query></span>"
  </div>
</template>
```

## State and methods

Like `select`, the controller exposes the `select-core` methods directly:

- `open()`
- `close()`
- `toggle()`
- `select(value)`
- `unselect(value)`
- `clear()`
- `toggleValue(value)`
- `highlight(index)`
- `highlightNext()`
- `highlightPrev()`
- `setSearch(query)`
- `getState()`
- `subscribe(listener)`

Example:

```js
autocomplete.subscribe((state) => {
  console.log(state.search, state.selectedValues);
});
```

## Lifecycle

The class API exposes `cleanup()`:

```js
const autocomplete = new Autocomplete('[data-select-input][data-autocomplete-id="framework-autocomplete"]');

// Later
autocomplete.cleanup();
```

The controller API still returns a `destroy()` function:

```js
const { destroy } = autocomplete.connect({
  element: '[data-select-input][data-autocomplete-id="framework-autocomplete"]',
});

// Later
destroy();
```

Use cleanup when the DOM can be replaced or unmounted.

## When to use Autocomplete vs Select

- Use autocomplete when the interaction starts from typing.
- Use [Select](/docs/components/select) when the interaction starts from a trigger button and option panel.
